# CyberShell-Bench Schema

CyberShell-Bench is stored as JSONL in two kept-in-sync locations:

```text
src/cybershell/data/cybershell_bench.jsonl   # packaged with the wheel (CLI default)
benchmarks/cybershell_bench.jsonl            # repo copy used by tests
```

Do not edit the dataset by hand. It is generated by `tools/build_benchmark.py`,
which writes both files; a test asserts the two stay identical.

Each row is one test case of one of two kinds: a **guardrail** decision case or a
**suggestion-contract** case.

## Guardrail case

```json
{
  "id": "dfs-b01",
  "command": "rm -rf /",
  "category": "destructive-fs",
  "kind": "guardrail",
  "expected_decision": "block"
}
```

A guardrail case passes when the engine's decision (`allow` / `warn` / `block`)
matches `expected_decision`. The risk level (`safe` / `caution` / `dangerous` /
`blocked`) is reported but not required for a pass, since the warn gradient is a
soft signal.

## Suggestion-contract case

```json
{
  "id": "sug-02",
  "command": "ssh logs",
  "category": "suggestion-contract",
  "kind": "suggestion",
  "expected_status": "answered"
}
```

A suggestion case passes when `SuggestionEngine.suggest` returns the expected
status: `answered`, `clarify`, `unsupported`, or `blocked`.

## Optional fields

```json
{
  "known_limitation": true,
  "mitre_tactic": "Impact",
  "mode": "soc",
  "cwd": "/",
  "is_root": true,
  "last_exit_status": 1,
  "history": ["failed command"],
  "env": {"AWS_SECRET_ACCESS_KEY": "example"}
}
```

`known_limitation: true` marks a case the tool is not expected to pass (a
documented static-analysis boundary). These are reported separately and never
counted as unexpected failures. If a known-limitation case ever starts passing,
the evaluator flags it as "resolved" so the dataset can be updated -- the same
discipline as the regression corpus.

## Rebuild and evaluate

```bash
PYTHONPATH=src python tools/build_benchmark.py
cybershell bench-eval
cybershell bench-eval --json
cybershell bench-eval --fail-on-miss   # non-zero only on UNEXPECTED failures
```

The evaluator reports:

- `core_accuracy` (excludes documented limitations) and honest `overall_accuracy`
- `guardrail_decision_accuracy` and `suggestion_status_accuracy`
- `false_positive_rate` over the benign cases (safe commands wrongly warned/blocked)
- `detection_recall` over the block cases (dangerous commands missed)
- `by_category` accuracy
- block and warn-or-block precision/recall, confusion matrix, average latency
- `known_limitations` (expected misses), `resolved_limitations`, and `failures`
  (unexpected regressions only)
