# Internationalization (i18n)

## Status: scaffold, not yet adopted

CyberShell Copilot is currently **English-only in practice**. This document and the
`cybershell.i18n` module define the *mechanism* for translation and a safe fallback,
so localization can be done incrementally rather than as a single large rewrite. The
rule messages and CLI output strings are **not yet routed through the helper**; doing
that migration is the remaining work.

This is deliberately honest: shipping a half-wired "i18n feature" that silently does
nothing would be worse than a clearly-scoped starting point.

## The mechanism

`cybershell.i18n.translate(key, **kwargs)` (aliased `_`) looks up a key in a JSON
catalog for the active locale and falls back to the key itself if the translation is
missing, so nothing ever crashes or blanks out.

```python
from cybershell.i18n import translate as _

_("status.answered")                       # English from locale/en.json
_("cli.blocked_prefix", reason="rm -rf /") # interpolation
```

- Active locale comes from the `CYBERSHELL_LOCALE` environment variable
  (default `en`).
- Catalogs live in `src/cybershell/locale/<locale>.json` and ship in the wheel.

## Migration plan (for a contributor)

1. **Externalize strings.** Replace user-facing literals in `cli.py` and the
   human-readable parts of risk/suggestion output with `_("some.key")`, adding the
   English text to `locale/en.json`.
2. **Decide on rule messages.** Guardrail `message` fields carry safety meaning.
   Either give each a stable key and translate the catalog, or keep rule messages in
   English and translate only the surrounding CLI framing. Translating safety-
   critical text needs review by someone fluent in the target language.
3. **Add a locale.** Copy `locale/en.json` to `locale/<lang>.json` and translate the
   values; set `CYBERSHELL_LOCALE=<lang>` to use it.
4. **Test.** Extend `tests/test_i18n.py` and assert that every key used in code
   exists in `locale/en.json` (a good lint to add once migration starts).

## Why it is deferred

Most operators of a security CLI work in English, and mistranslating a safety
message is itself a risk. So full i18n is intentionally a post-1.0, opt-in effort
with translation review — not something to rush. The scaffold ensures that effort
starts from a defined, tested foundation.
